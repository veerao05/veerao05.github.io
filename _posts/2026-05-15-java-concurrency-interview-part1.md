## Why Java Concurrency Matters

Java concurrency is one of the most tested topics in senior engineering interviews. Companies want to know if you can build systems that are both correct and performant under load.

## Core Concepts Covered in Part 1

- **Thread lifecycle** — States and transitions
- **Synchronisation** — synchronized keyword, intrinsic locks, and monitors
- **volatile keyword** — Memory visibility guarantees
- **Thread safety** — What it means and how to achieve it
- **Deadlock** — How to detect, prevent, and avoid it

## Common Interview Questions

1. What is the difference between synchronized and volatile?
2. How does Java's memory model affect multi-threaded programs?
3. What causes a deadlock, and how would you resolve one?
4. What is the difference between a process and a thread?
5. How does the ThreadLocal class work?
